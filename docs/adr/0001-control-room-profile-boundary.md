# Keep Conductor Generic And Put Local Experience In Profiles

Vault Conductor should own the generic lifecycle of agent work: Tasks, Runs, Workspaces, Human Gates, Review, tests, and PR handoff. The Obsidian plus cmux experience is a first-class Control Room Profile layered on top of that lifecycle, so the project can strongly support the intended control-room workflow without hardcoding one person's local workspace preferences into the core engine.
